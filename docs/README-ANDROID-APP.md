# Forum Personal Pod Android App

> Legacy note: this document describes the pre-edge `/api/civic/submit`
> Android path. The current app points at `secure-worker` and submits opt-in
> Forum Feedback to `/api/forum/feedback`, with member registration and
> aggregation handled in Cloudflare D1. Prefer the root `README.md` and
> `forum-airlock/docs/civic-edge-analysis.md` for current operations.

The Android app wraps `forum-pod` with Capacitor. It keeps the same local-first model:

- civic submissions are saved in the app WebView's IndexedDB;
- DuckDB hydrates from IndexedDB when the app opens;
- submissions sync to your cooperative server when online;
- failed submissions stay local and can be retried.

## Server URL

The Android app cannot use browser-relative `/api/civic/submit` unless it is hosted by the same web origin. After installing the app:

1. Open `Settings`.
2. Enter your cooperative server URL, for example:

   ```text
   https://pod.yourcommunity.forum
   ```

3. Tap `Save connection`.
4. Tap `Retry saved submissions now` if you already submitted offline.

The app will submit to:

```text
https://pod.yourcommunity.forum/api/civic/submit
```

## Build APK

Prerequisites:

- Node/npm. This project pins Capacitor 6 so Node 20 and Java 17 work.
- Java JDK 17. If `JAVA_HOME` points at `/usr/lib/jvm/java-17-openjdk-amd64` but that directory does not exist, unset it or let `build-android-apk.sh` auto-detect Temurin.
- Android Studio or Android SDK command-line tools
- Android platform/build tools installed

If a previous attempt installed Capacitor 8 and failed with `The Capacitor CLI requires NodeJS >=22.0.0`, or generated an Android project that fails with `invalid source release: 21`, clean the stale install first:

```bash
cd ~/Desktop/forum-pod
rm -rf node_modules package-lock.json android
npm install
npm install @capacitor/core@6 @capacitor/cli@6 @capacitor/android@6 --save-dev --save-exact
```

Build:

```bash
bash ~/Desktop/deploy/build-android-apk.sh
```

Output:

```text
~/Desktop/forum-pod/android/app/build/outputs/apk/debug/app-debug.apk
```

Install to a connected Android device:

```bash
adb install -r ~/Desktop/forum-pod/android/app/build/outputs/apk/debug/app-debug.apk
```

Or open the project in Android Studio:

```bash
cd ~/Desktop/forum-pod
npx cap open android
```

## Troubleshooting

- `bash: y: command not found`: you typed `y` at the shell prompt after the installer had already moved on. It is harmless.
- `Unable to locate package openjdk-17-jdk`: Java 17 is already installed on this machine via Temurin. You do not need Debian's `openjdk-17-jdk` package.
- `JAVA_HOME is set to an invalid directory`: run `unset JAVA_HOME`, or use `bash ~/Desktop/deploy/build-android-apk.sh`, which auto-detects the current Java path.
- `The Capacitor CLI requires NodeJS >=22.0.0`: stale Capacitor 8 is installed. Clean `node_modules` and `package-lock.json`, then run `npm install` so Capacitor 6 is installed from `package.json`.
- `invalid source release: 21`: the `android/` folder was generated by a newer Capacitor/Java-21 stack. Delete `android/`, keep Capacitor 6 in `package.json`, and rerun `bash ~/Desktop/deploy/build-android-apk.sh`.
- `cd: android: No such file or directory`: `npx cap add android` failed earlier. Fix the Node/Capacitor issue, then run `bash ~/Desktop/deploy/build-android-apk.sh`.

## App Store / Release APK

The generated `app-debug.apk` is for testing. For sharing widely, build a signed release APK or Android App Bundle from Android Studio.

## Important

- Phone-local data is local to that Android app install.
- If the app is uninstalled, its IndexedDB data is removed.
- The cooperative encrypted ledger still lives server-side in `forum_inbound.db`.
- AI chat on Android may not have WebGPU. Civic submission and local storage work regardless.
